## Pre-Interview Web Research (Required)

BEFORE BEGINNING QUESTIONS, research each identified vendor using web search:

For each vendor identified in Q13 (e.g., OpenAI, Salesforce Einstein, AWS Rekognition, etc.):

1. Search for the vendor's official **"terms of service"**, **"data processing agreement"**, **"acceptable use policy"**, and any **"AI usage policy"** pages.
2. Look specifically for:
   - Whether the vendor provides documented instructions for use (required under EU AI Act Art. 26(1))
   - What the vendor's logging and data retention policies state
   - Whether the vendor makes any AI transparency disclosures to end users
   - Whether the vendor's terms restrict certain deployment contexts (e.g., "not for use in healthcare decisions", "not for minors")
3. Also search for the **company's own website** for:
   - Any public statements about their use of third-party AI tools
   - Any user-facing disclosures about AI decision-making (privacy policy, cookie policy, AI disclosure pages)
   - Any worker-facing documentation (internal HR policies if publicly disclosed)

USE THIS RESEARCH to pre-fill answers where the public record is clear, and to flag discrepancies (e.g., _"Your privacy policy does not mention AI decision-making, but you are using [Vendor X] for [Purpose] — this may create an Art. 50 transparency gap"_).

## Questions (Q26–Q28) — Adaptive, Per Deployer System

### Q26. Have you received and reviewed the provider's instructions for use for [SYSTEM_NAME]?

**Options:**

- Yes — received and following
- Yes — received, not fully implemented
- No — never received
- N/A

**Agent Intelligence:**

Before asking, check the vendor website for published documentation, API docs, or usage guidelines.

- If formal documentation was found:

  > _"I found that [Vendor] publishes [specific document] — have you received and reviewed this as part of your deployment?"_

- If no formal docs were found:
  > _"I wasn't able to find formal instructions for use published by [Vendor]. Have they provided any deployment guidance directly to you?"_

**Critical Flag:** If answer is "No — never received":

> _"Under EU AI Act Art. 26(1), deployers are required to use AI systems in accordance with the provider's instructions. Not having received these creates a direct compliance exposure."_

---

### Q27. How do you retain logs generated by [SYSTEM_NAME]?

**Options:**

- All logs retained 6+ months
- Retained but \<6 months
- Some logs retained
- Don't retain logs
- System doesn't generate logs
- Not sure

**Agent Intelligence:**

Check vendor terms/documentation for what logs the system generates. If the vendor's docs specify log formats or retention policies:

> _"According to [Vendor]'s documentation, [SYSTEM_NAME] generates [X type of logs]. How are you currently retaining these?"_

**Critical Flag:** If answer is less than 6 months retention:

> _"EU AI Act Art. 26(6) requires a minimum 6-month log retention period. Your current approach may not meet this threshold."_

---

### Q28. Do you inform individuals that they are subject to decisions made by [SYSTEM_NAME]?

**Options:**

- Yes — all affected individuals notified
- Yes — some are notified
- No
- N/A

**Agent Intelligence:**

Cross-reference with Q20 (affected populations) and Q22 (potential consequences) from Stage 2. Also check:

- Company's public privacy policy for any AI decision-making disclosure language
- Company's employee-facing documentation (if Q20 includes Employees)
- Company's customer-facing terms for AI disclosure language

- If privacy policy already includes AI disclosure language:

  > _"I noticed your privacy policy mentions [X]. Does this cover all individuals affected by [SYSTEM_NAME], including [relevant group from Q20]?"_

- If no disclosure found anywhere:
  > _"I wasn't able to find any public disclosure about AI decision-making on your website. This is required under Art. 50 and Art. 26(7) for worker notification."_

**Critical Flag:** If Q20 includes "Employees" and answer is not "Yes — all":

> _"Art. 26(7) specifically requires notifying workers when they are subject to AI-based monitoring or decisions."_
